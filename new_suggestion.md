# 實驗指導原則與檢核表 (Experiment Guidelines & Action Items)

此文檔彙整了指導教授對於後續實驗的關鍵建議，請在執行各項實驗與數據分析時，務必遵守以下指導原則並逐一確認待辦事項。

## 🧠 第一部分：研究核心觀念與方向

### 1. 跳脫 AUC 迷思，針對「少數類」挑選正確的衡量指標

- **問題盲點**：在處理「破產預測」或「糖尿病預測」等極度不平衡的資料時，AUC 容易被數量龐大的大類（如正常公司、未患病者）拉高，無法真實反映模型對「少數類別（破產、患病）」的預測能力。
- **改進作法**：除了 AUC 外，應**重點關注 Type 1 Error（型一錯誤）、Recall（召回率）與 F1 Score**。對於金融或醫療等保守型機構而言，成功抓出少數類的代價極高，因此降低 Type 1 Error（將壞的預測成好的）往往比整體準確率更具業務價值。

### 2. 利用分數差異來創造研究的「故事性」

- **觀念轉折**：Baseline 的分數（如 AUC 偏低或 F1 Score 不理想）並非實驗失敗的壞事。
- **論文價值**：研究的價值在於**「找出既有方法的缺陷（例如對少數類預測極差），並用我們提出的創新方法去彌補與提升該特定指標」**。這才是整篇論文最核心的貢獻與故事脈絡。

### 3. 特徵選取 (Feature Selection) 的絕對鐵則

- **防範資料洩漏 (Data Leakage)**：進行特徵選取時，**絕對只能在訓練集（Train set）上進行**。嚴禁在包含測試集（Test set）的全量資料上做特徵選取，否則將導致嚴重的資料洩漏與過度擬合 (Overfitting)。

### 4. 確保實驗結果符合理論邏輯

- **邏輯自洽**：理論上，加入適當的 Sampling（解決資料不平衡）或 Feature Selection（過濾雜訊）後，模型表現應該要比 Baseline 更好。
- **異常排查**：如果實驗結果沒有變好，**必須先回頭檢查程式碼實作與實驗流程（Pipeline）是否有 Bug 或邏輯瑕疵**。唯有在確認程式碼 100% 無誤的前提下，才能推論這是特定資料集本身的特性。

## ✅ 第二部分：具體待辦任務清單 (Next Action Items)

以下清單為下次與教授會議前需完成的重點任務，請於執行後打勾確認：

### 任務一：完整擴充並記錄評估指標

- [ ] 針對已經跑完的 Baseline 模型，重新撈取並記錄詳細指標。
- [ ] 程式碼驗證報告需確保輸出 **Precision、Recall、F1 Score、Type 1 Error (False Positive Rate)** 等指標，供後續比較與敘事備用。

### 任務二：實作並驗證資料不平衡採樣機制 (Sampling)

- [ ] 於 Baseline 基礎上，加入資料採樣機制以處理類別不平衡問題。
- [ ] 確實分別實作以下三種策略：**Under-sampling（欠採樣）**、**Over-sampling（過採樣）** 以及 **Hybrid（混合採樣）**。
- [ ] 對比驗證：檢視採樣後的各項指標（特別是 F1 與 Recall）是否有比未採樣的 Baseline 更好。

### 任務三：嚴謹落實時序交叉驗證與特徵選取流程

- [ ] 實作「五折交叉驗證 (5-fold Cross-Validation)」，且**必須嚴格遵守時間序列原則**（明確保留時間先後順序，例如：Train Old、Test New，嚴禁未來資料混入過去）。
- [ ] 程式碼檢查：確認 `SelectKBest` 等特徵選取演算法，**僅作用於劃分好的 Train data**，再將選出的特徵規則套用至 Test data 進行驗證評估。

### 任務四：製作綜合比較表格與視覺化 CSV

- [ ] 將分散的 CSV 結果彙整為一張「一眼就能看出優劣」的綜合對照表。
- [ ] 表格需涵蓋以下關鍵維度：
  - **資料集 (Datasets)**：破產 (Bankruptcy)、股票 (Stock)、糖尿病 (Medical)。
  - **訓練策略 (Strategies)**：Baseline (Retrain 全資料)、Fine-tune 最新資料等。
  - **模型處置方法**：未採樣 vs. (Under/Over/Hybrid) 採樣；無特徵選取 vs. 有特徵選取。
  - **關鍵績效指標 (KPIs)**：至少並列放入 **AUC** 與 **F1 Score** 等重點指標。
- [ ] 確保對照表排版清晰，以便下次開會時迅速向教授展示各種組合的差異與亮點結論。
